## 1. AI 概述与历史演进

### 1.1 什么是人工智能

**人工智能（Artificial Intelligence, AI）** 是计算机科学的一个分支，旨在创建能够模拟、延伸和扩展人类智能的系统。其核心目标是让机器具备**感知、推理、学习、决策和创造**的能力。

从能力层级来看，AI 可分为三个层次：

| 层级 | 名称 | 说明 |
|------|------|------|
| **弱 AI (Narrow AI)** | 狭义人工智能 | 在特定任务上达到或超越人类的系统（如 AlphaGo、ChatGPT）。当前所有 AI 均属此类。 |
| **强 AI (General AI / AGI)** | 通用人工智能 | 具备与人类同等的通用认知能力，能解决任意智力任务。尚未实现。 |
| **超 AI (Super AI / ASI)** | 超级人工智能 | 在所有领域全面超越人类的智能。纯理论阶段。 |

### 1.2 AI 的子领域
```
人工智能 (AI)
├── 机器学习 (Machine Learning)
│ ├── 监督学习 (Supervised Learning)
│ ├── 无监督学习 (Unsupervised Learning)
│ ├── 半监督学习 (Semi-supervised Learning)
│ ├── 自监督学习 (Self-supervised Learning)
│ └── 强化学习 (Reinforcement Learning)
├── 深度学习 (Deep Learning) — 机器学习的一个子集
│ ├── CNN (卷积神经网络)
│ ├── RNN/LSTM (循环神经网络)
│ ├── Transformer
│ ├── GAN (生成对抗网络)
│ └── Diffusion Models (扩散模型)
├── 自然语言处理 (NLP)

├── 语音识别与合成 (Speech)
├── 机器人学 (Robotics)

```

## 2. 数学基础

> AI 的三大数学支柱：**线性代数**（数据表示）、**概率论与统计**（不确定性建模）、**微积分与优化**（模型训练）。

### 2.1 线性代数

#### 2.1.1 标量、向量、矩阵、张量

| 概念 | 维度 | 表示 | 示例 |
|------|------|------|------|
| **标量 (Scalar)** | 0 维 | a ∈ ℝ | 3.14, 温度值 |
| **向量 (Vector)** | 1 维 | x ∈ ℝ^n | 一个样本的特征：[身高, 体重, 年龄] |
| **矩阵 (Matrix)** | 2 维 | X ∈ ℝ^{m × n} | 一批 m 个样本，每个 n 个特征 |
| **张量 (Tensor)** | ≥3 维 | 𝓧 ∈ ℝ^{d_1 × d_2 × ...} | 图像 (H×W×C)、视频 (T×H×W×C) |

**原理**：在深度学习中，数据几乎总是以张量形式存在。例如：一张 RGB 图像是 `[3, 224, 224]` 的张量；一个 batch 的文本 token 是 `[batch_size, seq_len, hidden_dim]` 的张量。所有神经网络操作——矩阵乘法、卷积、注意力——本质上都是张量运算。

#### 2.1.2 矩阵运算

**矩阵乘法**（深度学习的核心运算）：
```math
C = A × B, A ∈ ℝ^{m × k}, B ∈ ℝ^{k × n}
```

神经网络的前向传播本质上就是一连串的矩阵乘法 + 非线性激活函数：
```math
h = σ(W x + b)
```

其中 W 是权重矩阵，x 是输入向量，b 是偏置，σ 是激活函数。

**点积 (Dot Product)**：a · b = ∑_i a_i b_i = ‖a‖ ‖b‖ cos θ。用于计算注意力机制中的相似度分数。

**Hadamard 积 (逐元素乘法)**：a ⊙ b。用于 LSTM 门控机制和 Dropout。

**转置 (Transpose)**：A^T_{ij} = A_{ji}。用于梯度计算中调整维度匹配。

#### 2.1.3 特征值与特征向量

如果 Av = λ v，则 v 是 A 的特征向量，λ 是对应的特征值。

**在 AI 中的应用**：
- **PCA（主成分分析）**：通过对协方差矩阵进行特征值分解，找到数据方差最大的方向（主成分），从而实现降维。
- **谱聚类**：利用图的拉普拉斯矩阵的特征向量进行聚类。
- **矩阵条件数**：最大与最小特征值之比，反映优化问题的难易程度。

#### 2.1.4 奇异值分解 (SVD)
```math
A = U Σ V^T
```

- U：左奇异向量矩阵（列空间）
- Σ：奇异值对角矩阵（按降序排列）
- V：右奇异向量矩阵（行空间）

**在 AI 中的应用**：
- **矩阵压缩/低秩近似**：取前 k 个最大奇异值，实现模型压缩（如 LoRA 使用低秩分解微调大模型）。
- **推荐系统**：SVD++ 用于协同过滤。
- **词向量降维**：对词-上下文矩阵做 SVD 得到稠密词表示（早期的词嵌入方法）。

### 2.2 概率论与统计

#### 2.2.1 基本概念

| 概念 | 定义 | AI 中的含义 |
|------|------|------------|
| **随机变量** | 取值不确定的变量 | 模型输出、噪声 |
| **概率分布** | P(X=x) | 分类器输出的类别概率 |
| **条件概率** | P(Y | X) | 给定输入 X 时标签 Y 的概率 |
| **联合概率** | P(X,Y) | 输入和标签同时发生的概率 |
| **边缘概率** | P(X) = ∑_Y P(X,Y) | 输入数据本身的分布 |
| **贝叶斯定理** | P(Y | X) = (P(X | Y)P(Y))/(P(X)) | 从先验 + 似然推导后验 |

#### 2.2.2 贝叶斯定理 — AI 推理的基石
```math
P(H|E) = (P(E|H) · P(H))/(P(E))
```

- P(H)：**先验概率** — 在看到数据之前对假设的信念
- P(E|H)：**似然** — 在假设为真时观察到数据的概率
- P(H|E)：**后验概率** — 看到数据后更新的信念
- P(E)：**证据** — 归一化常数

**应用**：
- **朴素贝叶斯分类器**：假设特征条件独立，P(y|x_1,...,x_n) ∝ P(y)∏_i P(x_i|y)
- **贝叶斯神经网络**：对权重施加先验分布，输出不确定性估计
- **变分自编码器 (VAE)**：用变分推断学习潜在变量的后验分布

#### 2.2.3 信息论基础

**熵 (Entropy)** — 衡量随机变量的不确定性：
```math
H(X) = -∑_x P(x) log P(x)
```

熵越大，不确定性越高。熵为 0 表示完全确定。

**交叉熵 (Cross-Entropy)** — 衡量两个分布的距离：
```math
H(P, Q) = -∑_x P(x) log Q(x)
```

**KL 散度 (Kullback-Leibler Divergence)** — 衡量分布 Q 与 P 的差异：
```math
D_{KL}(P ∥ Q) = ∑_x P(x) log (P(x))/(Q(x))
```

**在 AI 中的核心应用**：
- **交叉熵损失** = 分类任务的标准损失函数：𝓛 = -∑_c y_c log ŷ_c
- **KL 散度**：VAE 中约束潜在分布接近标准正态分布；RLHF 中约束策略更新幅度
- **互信息**：特征选择、表示学习中的 InfoNCE 损失

### 2.3 微积分与优化

#### 2.3.1 导数与梯度

**导数**定义：f'(x) = lim_{h → 0} (f(x+h) - f(x))/(h)，表示函数在某点的瞬时变化率。

**偏导数**：多元函数对某一变量的导数，其他变量视为常数。

**梯度 (Gradient)**：所有偏导数组成的向量 ∇ f = [(∂ f)/(∂ x_1), ..., (∂ f)/(∂ x_n)]^T。

梯度指向函数增长最快的方向。因此，**梯度下降**沿负梯度方向更新参数以最小化损失：
```math
θ_{t+1} = θ_t - η ∇_θ 𝓛(θ_t)
```

#### 2.3.2 链式法则与反向传播

**链式法则**是多层复合函数求导的核心：
```math
(dz)/(dx) = (dz)/(dy) · (dy)/(dx)
```

**反向传播 (Backpropagation)** 就是将链式法则系统化地应用于神经网络。计算图上的每个节点只需知道**局部梯度**和**上游梯度**→两者相乘 → 传递给下游。这使得计算所有参数的梯度只需一次前向 + 一次反向，复杂度与参数量成正比。

具体过程见第 4.3 节。

#### 2.3.3 优化方法

详见第 4.5 节。核心思想演进：
```
梯度下降 (GD) → 随机梯度下降 (SGD) → SGD + Momentum
 ↓
 AdaGrad → RMSprop → Adam (最常用)
```

## 3. 传统机器学习

> 传统机器学习是深度学习的基础，许多思想（偏差-方差权衡、正则化、集成）在深度学习中仍然核心。

### 3.1 机器学习基本范式

#### 3.1.1 监督学习 (Supervised Learning)

**定义**：给定带标签的数据集 𝓓 = {(x_i, y_i)}_{i=1}^N，学习映射 f: 𝓧 → 𝓨。

**分类 (Classification)**：y 是离散类别
- 二分类：垃圾邮件检测（y ∈ {0, 1}）
- 多分类：图像分类（y ∈ {猫, 狗, 鸟, ...}）

**回归 (Regression)**：y 是连续值
- 房价预测、股票价格预测

#### 3.1.2 无监督学习 (Unsupervised Learning)

**定义**：只有输入 𝓓 = {x_i}_{i=1}^N，没有标签。目标是发现数据的内在结构。

- **聚类 (Clustering)**：K-Means, DBSCAN, 层次聚类
- **降维 (Dimensionality Reduction)**：PCA, t-SNE, UMAP
- **密度估计 (Density Estimation)**：GMM, 核密度估计

#### 3.1.3 半监督学习

少量标签 + 大量无标签数据。利用无标签数据改善模型性能。常见方法：
- **自训练 (Self-training)**：用模型预测伪标签
- **一致性正则化**：对扰动输入鼓励一致输出

#### 3.1.4 自监督学习

**定义**：从数据本身构造监督信号，无需人工标签。这是当前大模型预训练的核心范式。

示例：
- **掩码语言模型 (MLM)**：随机遮盖文本中的词，让模型预测被遮盖的词（BERT）
- **下一句预测 (NSP)**：判断两个句子是否相邻
- **对比学习 (Contrastive Learning)**：让相似样本的表示接近，不相似样本的表示远离（SimCLR, CLIP）
- **下一 token 预测 (Next Token Prediction)**：GPT 的核心训练任务

### 3.2 核心概念

#### 3.2.1 偏差-方差权衡 (Bias-Variance Tradeoff)

这是理解模型泛化能力的最核心框架之一。

**数学分解**：对于平方损失，期望预测误差可分解为：
```math
𝔼[(y - f̂(x))^2] = Bias[f̂(x)]^2 + Var[f̂(x)] + σ^2
```

| 高偏差 (欠拟合) | 高方差 (过拟合) |
|:---:|:---:|
| 模型太简单 | 模型太复杂 |
| 训练误差高 | 训练误差低但测试误差高 |
| 解决方法：增加模型复杂度 | 解决方法：更多数据、正则化、集成 |

**演进脉络**：传统观点认为偏差和方差不可兼得（U 形曲线）。但深度学习时代发现，超大模型在"插值阈值"之后，测试误差继续下降——这就是**双下降 (Double Descent)** 现象，挑战了经典观点。

#### 3.2.2 过拟合与正则化

**过拟合**：模型记住了训练数据中的噪声而非学习到真实规律。

**正则化方法**：
- **L1 正则化 (Lasso)**：𝓛_{total} = 𝓛 + λ ∑ |w_i| → 产生稀疏解
- **L2 正则化 (Ridge/权重衰减)**：𝓛_{total} = 𝓛 + λ ∑ w_i^2 → 约束权重大小
- **早停 (Early Stopping)**：验证误差不再下降时停止训练
- **Dropout**：随机丢弃神经元（详见第 4 章）
- **数据增强**：对训练数据做随机变换以增加多样性

#### 3.2.3 模型评估

**分类指标**：

| 真实 / 预测 | 正 | 负 |
|:---:|:---:|:---:|
| 正 | TP | FN |
| 负 | FP | TN |

- **准确率** = (TP+TN)/(TP+TN+FP+FN)
- **精确率 (Precision)** = (TP)/(TP+FP) — 预测为正中有多少是真
- **召回率 (Recall)** = (TP)/(TP+FN) — 真实正例中被找出多少
- **F1-Score** = 2 · (P · R)/(P+R) — 精确率和召回率的调和平均
- **ROC-AUC**：ROC 曲线下面积，衡量模型区分正负类的能力
- **混淆矩阵**：直观展示各类别的分类情况

**回归指标**：
- **MSE (均方误差)** = (1)/(n)∑ (y_i - ŷ_i)^2
- **MAE (平均绝对误差)** = (1)/(n)∑ |y_i - ŷ_i|
- **R^2 (决定系数)** = 1 - (SS_{res})/(SS_{tot})

**交叉验证 (Cross-Validation)**：将数据分为 K 折，轮换验证集，取平均性能。K=5 或 10 最常用。

### 3.3 经典算法详解

#### 3.3.1 线性回归 (Linear Regression)

**模型**：y = w^T x + b = ∑_{i=1}^d w_i x_i + b

**原理**：假设目标值与特征之间存在线性关系。目标是找到一组权重 w 使得预测值与真实值的均方误差最小。

**闭式解 (Normal Equation)**：
```math
w^* = (X^T X)^{-1} X^T y
```

#### 3.3.2 逻辑回归 (Logistic Regression)

虽然名字叫"回归"，但它是**分类**模型。

**原理**：将线性函数的输出通过 Sigmoid 函数映射到 (0, 1) 区间，解释为概率。
```math
P(y=1|x) = σ(w^T x + b) = (1)/(1 + e^{-(w^T x + b))}
```

**损失函数**（交叉熵）：
```math
𝓛 = -(1)/(N)∑_{i=1}^N [y_i log(ŷ_i) + (1-y_i) log(1-ŷ_i)]
```

**决策边界**：w^T x + b = 0，即 P(y=1|x) = 0.5 的位置。它是一个**线性超平面**。

**多分类扩展 (Softmax 回归)**：
```math
P(y=k|x) = (e^{w_k^T x})/(∑_{j=1}^K e^{w_j^T x})
```

#### 3.3.3 支持向量机 (SVM)

**核心思想**：找到一个超平面，使得它到最近样本点（支持向量）的**间隔 (Margin)** 最大化。

**原理演进**：

1. **硬间隔 SVM**（数据线性可分时）：
```math
min_{w, b} (1)/(2) ‖w‖^2 s.t. y_i(w^T x_i + b) ≥ 1
```

2. **软间隔 SVM**（引入松弛变量，允许少量错误分类）：
```math
min_{w, b, ξ} (1)/(2) ‖w‖^2 + C∑_i ξ_i
```

 其中 C 控制间隔宽度与错误容忍的权衡。

3. **核技巧 (Kernel Trick)**：通过核函数 K(x_i, x_j) = φ(x_i)^T φ(x_j) 隐式地将数据映射到高维空间，使其线性可分，而无需显式计算高维映射。

常见核函数：
- **线性核**：K(x, x') = x^T x'
- **RBF 核**：K(x, x') = exp(-γ ‖x - x'‖^2)
- **多项式核**：K(x, x') = (x^T x' + c)^d

**对偶问题**：利用拉格朗日对偶性，SVM 的优化可转化为只依赖内积的形式，这是使用核技巧的关键。

#### 3.3.4 决策树 (Decision Tree)

**原理**：递归地将数据空间划分为多个矩形区域，每个区域预测一个常数值（回归）或多数类别（分类）。

**分裂准则**：
- **信息增益**（ID3）：IG(D, A) = H(D) - ∑_v (|D_v|)/(|D|) H(D_v)，选择使信息增益最大的特征
- **增益率**（C4.5）：GainRatio(D, A) = (IG(D, A))/(-∑ |D_v|/|D|log(|D_v|)/(|D|)}，解决信息增益偏向多值特征的问题
- **基尼指数**（CART）：Gini(D) = 1 - ∑_k p_k^2，选择基尼指数最小的划分

**优缺点**：
- 优点：可解释性强，不需要特征缩放，自动特征选择
- 缺点：容易过拟合，不稳定（数据微小变化可能导致树结构大幅改变）

#### 3.3.5 随机森林 (Random Forest)

**原理**：集成多棵决策树，通过投票（分类）或平均（回归）得到最终预测。

**两个随机性来源**（降低方差的秘诀）：
1. **Bootstrap 采样**：每棵树用有放回抽样得到不同训练集
2. **特征随机子集**：每次分裂时只考虑随机选择的一部分特征

**为什么有效**：假设每棵树的方差为 σ^2，树之间的相关性为 ρ，则集成后的方差为：
```math
Var(ensemble) = ρ σ^2 + (1-ρ)/(B) σ^2
```

增加树的数量 B 可降低第二项，而特征随机子集降低了 ρ，双重降方差。

**Out-of-Bag (OOB) 误差**：每棵树只用约 63.2% 的样本，剩余的 36.8% 可作为该树的验证集，提供免费的无偏估计。

#### 3.3.6 梯度提升 (Gradient Boosting)

**演进脉络**：AdaBoost → Gradient Boosting → XGBoost → LightGBM → CatBoost

**核心原理**：与 Bagging（并行训练多棵树取平均）不同，Boosting **串行**训练：每棵新树拟合前面所有树的**残差**（负梯度方向）。

数学上，第 t 步的加法模型是：
```math
F_t(x) = F_{t-1}(x) + η · h_t(x)
```

其中 h_t(x) 拟合残差 -[(∂ 𝓛(y_i, F(x_i)))/(∂ F(x_i))]_{F=F_{t-1}}，η 是学习率。

**XGBoost 的改进**：在目标函数中显式加入正则项，使用二阶泰勒展开（用到 Hessian 信息），支持列采样和并行特征计算。

**LightGBM**：使用基于直方图的算法和 GOSS (Gradient-based One-Side Sampling)，大幅提升训练速度，适合大数据集。

#### 3.3.7 K-近邻 (KNN)

**原理**：**惰性学习 (Lazy Learning)**——不显式训练模型，预测时找训练集中最近的 K 个邻居，投票决定。

**关键要素**：
- **距离度量**：欧氏距离、曼哈顿距离、余弦相似度
- **K 值选择**：K 太小 → 过拟合（对噪声敏感）；K 太大 → 欠拟合（决策边界过于平滑）
- **决策规则**：多数投票（分类）、距离加权平均（回归）

**Curse of Dimensionality (维数灾难)**：高维空间中所有点之间的距离趋于相等，KNN 失效。因此 KNN 通常只适用于低维数据。

#### 3.3.8 K-Means 聚类

**原理**：将数据分为 K 个簇，最小化簇内平方和（Within-Cluster Sum of Squares, WCSS）：
```math
argmin_S ∑_{k=1}^K ∑_{x_i ∈ S_k} ‖x_i - μ_k‖^2
```

**Lloyd 算法**（EM 视角）：
1. **E 步 (Assignment)**：将每个点分配给最近的质心
2. **M 步 (Update)**：重新计算每个簇的质心（均值）
3. 迭代至收敛

**注意事项**：
- K 值需预先指定（可用肘部法则或轮廓系数选择）
- 对初始质心敏感 → 使用 K-Means++ 初始化
- 只适用于凸簇，对异常值敏感

#### 3.3.9 PCA (主成分分析)

**原理**：寻找数据方差最大的正交方向（主成分），将数据投影到低维空间。

**数学推导**：
1. 中心化数据：X ← X - x̄
2. 计算协方差矩阵：C = (1)/(n-1) X^T X
3. 特征值分解：C = V Λ V^T
4. 取前 k 个最大特征值对应的特征向量作为投影矩阵 V_k
5. 降维：Z = X V_k

**方差解释率**：(∑_{i=1}^k λ_i)/(∑_{i=1}^d λ_i)，帮助决定保留多少主成分。

## 4. 深度学习基础

> 深度学习 = 多层非线性变换 + 基于梯度的端到端优化。

### 4.1 人工神经元

**生物学启发**：神经元接收来自其他神经元的信号，当累计信号超过阈值时"激活"并向下传递信号。

**数学模型**（McCulloch-Pitts 神经元, 1943）：
```math
y = σ(∑_{i=1}^n w_i x_i + b) = σ(w^T x + b)
```

- x：输入信号（来自上一层的输出或原始数据）
- w：**权重 (Weights)** — 每个连接的重要性，是学习的目标
- b：**偏置 (Bias)** — 控制激活的阈值
- σ：**激活函数** — 引入非线性

### 4.2 激活函数

> 没有激活函数，多层网络等价于单层线性模型。激活函数引入非线性，赋予网络表达能力。

| 函数 | 公式 | 特点 |
|------|------|------|
| **Sigmoid** | σ(x) = (1)/(1+e^{-x)} | 输出 (0,1)，有梯度消失问题；用于二分类输出层 |
| **Tanh** | tanh(x) = (e^x-e^{-x})/(e^x+e^{-x}) | 输出 (-1,1)，零中心化，但仍梯度消失 |
| **ReLU** | ReLU(x) = max(0, x) | 计算简单，缓解梯度消失，最常用；缺点：神经元可能"死亡" |
| **Leaky ReLU** | max(α x, x) | 修复 ReLU 的死亡问题 |
| **GELU** | x · Φ(x) | Transformer 标配，比 ReLU 平滑 |
| **Swish/SiLU** | x · σ(x) | 自门控，某些任务优于 ReLU |
| **Softmax** | (e^{x_i})/(∑_j e^{x_j}) | 将 logits 转化为概率分布，用于多分类输出 |

**ReLU 为什么能缓解梯度消失？**
Sigmoid/Tanh 的导数最大值分别为 0.25 和 1，深层网络中层间连乘导致梯度指数衰减。ReLU 在正区间的导数为 1，梯度可以无损地反向传播。

**GELU 的原理**（Transformer 标准激活函数）：
```math
GELU(x) = x · P(X ≤ x), X ∼ 𝓝(0, 1)
```

直觉：以概率 x 为负时"关闭"神经元，结合了 ReLU 的稀疏性和 Dropout 的正则化思想。

### 4.3 多层感知机 (MLP) 与反向传播

#### 4.3.1 MLP 结构
```
输入层 隐藏层 输出层
 x₁ ────→ h₁ ────→
 ╲ ╱ │ ╲ ╱
 ╳ │ ╳ → ŷ
 ╱ ╲ │ ╱ ╲
 x₂ ────→ h₂ ────→
```

每个隐藏层：
```math
h^{(l)} = σ(W^{(l)} h^{(l-1)} + b^{(l)})
```

**通用近似定理 (Universal Approximation Theorem)**：一个具有足够多隐藏神经元（含非线性激活函数）的单隐层网络可以以任意精度逼近任何连续函数。这解释了为什么神经网络理论上非常强大。

#### 4.3.2 反向传播 (Backpropagation) — 深度学习的根基

**问题**：如何高效计算损失函数对每个参数的梯度？

**解法**：利用链式法则，从输出层向输入层逐层计算。

**四步过程**：

1. **前向传播 (Forward Pass)**：计算每一层的输出 h^{(l)} 并缓存
2. **计算输出层梯度**：δ^{(L)} = ∇_{h^{(L)}} 𝓛 ⊙ σ'(z^{(L)})
3. **反向传播 (Backward Pass)**：对 l = L-1, ..., 1：
```math
δ^{(l)} = (W^{(l+1)})^T δ^{(l+1)} ⊙ σ'(z^{(l)})
```

4. **参数梯度**：∇_{W^{(l)}} 𝓛 = δ^{(l)} (h^{(l-1)})^T

**时间复杂度**：𝓞(N_{params})，与参数量成正比，这是深度学习可行性的关键。

**为什么深度学习直到 2010s 才爆发？**
- **数据**：互联网产生了海量标注数据（如 ImageNet）
- **算力**：GPU 使大规模矩阵运算成为可能
- **算法**：ReLU、BatchNorm、ResNet 等解决了深层网络的训练问题

### 4.4 损失函数

| 任务 | 损失函数 | 公式 |
|------|----------|------|
| 二分类 | Binary Cross-Entropy | -[ylogŷ + (1-y)log(1-ŷ)] |
| 多分类 | Categorical Cross-Entropy | -∑_c y_c log ŷ_c |
| 回归 | MSE | (1)/(n)∑(y_i - ŷ_i)^2 |
| 回归（对异常值鲁棒） | MAE / Huber | 介于 MSE 与 MAE 之间 |
| 对比学习 | InfoNCE | -log(exp(sim(q,k_+)/τ))/(∑_{k)exp(sim(q,k)/τ)} |

### 4.5 优化算法

#### 4.5.1 演进脉络
```
SGD (朴素)
 │
 ├──→ SGD + Momentum (加入惯性，加速收敛)
 │ v_{t+1} = βv_t - η∇L(θ_t)
 │ θ_{t+1} = θ_t + v_{t+1}
 │
 ├──→ AdaGrad (自适应学习率，每个参数有不同的学习率)
 │ 缺点：学习率单调递减至 0
 │
 ├──→ RMSprop (用指数移动平均替代累加，解决 AdaGrad 的衰减问题)
 │
 └──→ Adam (结合 Momentum + RMSprop，最常用)
 m_t = β₁m_{t-1} + (1-β₁)g_t ← 一阶矩（动量）
 v_t = β₂v_{t-1} + (1-β₂)g_t² ← 二阶矩（自适应学习率）
 m̂_t = m_t/(1-β₁^t), v̂_t = v_t/(1-β₂^t) ← 偏差校正
 θ_t = θ_{t-1} - η · m̂_t/(√v̂_t + ε) ← 参数更新
```

**Adam 为什么好用**：
- 动量项加速收敛，减少震荡
- 自适应学习率使得对稀疏特征有更大步长
- 偏差校正解决了训练初期的冷启动问题
- 对大多数超参数不敏感，开箱即用

**AdamW**：将权重衰减（L2 正则化）从自适应学习率中解耦出来，是现代训练的标配：
```math
θ_t = θ_{t-1} - η · (m̂_t)/(√(v̂_t) + ε) - η λ θ_{t-1}
```

#### 4.5.2 学习率调度

| 策略 | 描述 |
|------|------|
| **Step Decay** | 每 N 个 epoch 将学习率乘以衰减因子 |
| **Cosine Annealing** | η_t = η_{min} + (1)/(2)(η_{max} - η_{min})(1 + cos((t)/(T)π)) |
| **Warmup** | 前几个 step 线性增加学习率，避免训练初期不稳定 |
| **ReduceLROnPlateau** | 验证损失不下降时降低学习率 |

**Warmup 的必要性**：训练开始时，模型参数是随机的，梯度的方向和幅度都不稳定。如果一开始就用大学习率，可能导致训练发散。Warmup 让模型先用小步长找到 "初始方向"。

### 4.6 正则化与归一化

#### 4.6.1 Dropout

**原理**：训练时以概率 p 随机"丢弃"神经元（输出设为 0），测试时不丢弃。

**为什么有效**？
1. **集成视角**：每次 Dropout 相当于训练一个不同的子网络，测试时近似为 2^n 个子网络的平均
2. **防止共适应**：神经元不能依赖特定其他神经元的存在

#### 4.6.2 Batch Normalization (BN)

**原理**：对每个 mini-batch 的激活值做标准化：
```math
x̂ = (x - μ_B)/(σ_B), y = γ x̂ + β
```

其中 μ_B, σ_B 是 batch 内的均值和方差，γ, β 是可学习的缩放和偏移参数。

**好处**：
1. 允许使用更大的学习率，加速收敛
2. 降低对初始化的敏感度
3. 有一定的正则化效果（batch 的随机性引入噪声）
4. 缓解内部协变量偏移 (Internal Covariate Shift)

#### 4.6.3 Layer Normalization (LN)

**原理**：与 BN 不同，LN 在**特征维度**上做标准化（而非 batch 维度），因此不依赖 batch 大小。
```math
μ = (1)/(H)∑_{i=1}^H x_i, σ = √((1)/(H)∑_{i=1)^H (x_i - μ)^2}
```

**LN 是 Transformer 的核心组件**，因为 NLP 中序列长度不一，batch 统计不稳定。每个样本独立标准化，更适合序列建模。

## 5. 卷积神经网络 (CNN) 与计算机视觉

> CNN 是受生物视觉皮层启发的架构，利用局部连接和权重共享高效处理网格结构数据。

### 5.1 核心操作

#### 5.1.1 卷积层 (Convolution Layer)

**原理**：使用可学习的**滤波器 (Filter/Kernel)** 在输入上滑动，计算局部区域的加权和。
```math
(I * K)[i, j] = ∑_m ∑_n I[i+m, j+n] · K[m, n]
```

**为什么卷积有效？**
1. **稀疏连接 (Sparse Connectivity)**：每个神经元只与输入的一小块区域连接。参数从 H × W × H × W 降至 K × K。
2. **参数共享 (Weight Sharing)**：同一个滤波器在整个图像上使用相同的权重，检测相同特征无论其出现在哪里。
3. **平移等变性 (Translation Equivariance)**：输入平移导致输出同样平移，这对图像理解很自然。

#### 5.1.2 池化层 (Pooling Layer)

**最大池化 (Max Pooling)**：取局部窗口内的最大值
**平均池化 (Avg Pooling)**：取局部窗口内的平均值

**作用**：
- 降低特征图的空间尺寸（减少计算量）
- 获得局部平移不变性
- 扩大感受野

#### 5.1.3 感受野 (Receptive Field)

**定义**：输出特征图上一个像素对应输入图像上的区域大小。

**计算**（递推公式）：
```math
RF_{l} = RF_{l-1} + (K_l - 1) × ∏_{i=1}^{l-1} S_i
```

其中 K_l 是卷积核大小，S_i 是步长。深层神经元能看到更大的输入区域，这是 CNN 层次化特征提取的基础。

### 5.2 经典架构演进
```
LeNet (1998)
 │ 最早的实用 CNN，手写数字识别。Conv → Pool → Conv → Pool → FC
 │
AlexNet (2012) — ImageNet 冠军，深度学习革命的起点
 │ 更深的网络、ReLU、Dropout、GPU 并行训练
 │
VGG (2014) — 证明了深度的重要性
 │ 全部使用 3×3 卷积堆叠，简洁统一
 │
GoogLeNet/Inception (2014) — 多尺度并行卷积
 │ 1×1、3×3、5×5 卷积并行，不同感受野捕捉不同尺度特征
 │
ResNet (2015) — 残差连接，突破深度瓶颈
 │ 跳跃连接 (Skip Connection)：H(x) = F(x) + x
 │
DenseNet (2017) — 极致连接
 │ 每层与所有前层相连，特征复用最大化
 │
EfficientNet (2019) — 用 NAS 搜索最优的深度/宽度/分辨率比例
 │
ViT (2020) — Transformer 进入视觉
 │ 将图像切为 patch，当作 token 输入 Transformer
 │
ConvNeXt (2022) — "现代化"的 CNN，融入 Transformer 设计思想
```

#### 5.2.1 ResNet — 里程碑

**核心问题**：网络越深，训练误差反而越高（不是过拟合，是优化困难导致的**退化问题**）。

**解决方案 — 残差学习**：
```math
y = 𝓕(x, {W_i}) + x
```

学习目标是残差 𝓕(x) = H(x) - x 而非直接学 H(x)。如果恒等映射是最优的，网络可以轻松地让 𝓕(x) → 0。

**为什么能训练超过 100 层？**
- 梯度可以通过跳跃连接直接回传，避免了梯度消失：(∂ 𝓛)/(∂ x) = (∂ 𝓛)/(∂ y) · (1 + (∂ 𝓕)/(∂ x))，常数项 1 保证了梯度不会消失。

### 5.3 目标检测

#### 5.3.1 两阶段检测器

**R-CNN 系列**：
1. **R-CNN**：选择性搜索 → 区域提案 → 每个区域跑 CNN → SVM 分类。慢。
2. **Fast R-CNN**：整图 CNN → RoI Pooling → 分类+回归。快很多。
3. **Faster R-CNN**：引入 RPN (Region Proposal Network) 在特征图上直接生成候选框，实现端到端训练。

**核心 — RoI Pooling**：将任意大小的候选区域映射到固定大小的特征表示，使得后续全连接层可以处理。

#### 5.3.2 单阶段检测器

**YOLO 系列**（演进：YOLOv1 → ... → YOLOv11）：
- 将检测视为回归问题：直接在图像网格上预测边界框和类别
- 快但早期精度不如两阶段，v3 之后逐渐持平
- **核心原理**：将图像分为 S×S 网格，每个网格预测 B 个边界框 + 类别概率

**对比**：

| 特性 | 两阶段 (Faster R-CNN) | 单阶段 (YOLO) |
|------|----------------------|--------------|
| 速度 | 慢 (≈5-10 FPS) | 快 (≈30-100+ FPS) |
| 精度 | 高 | 中/高 (逐渐追平) |
| 适用场景 | 高精度需求 | 实时场景 |

### 5.4 图像分割

**语义分割**：为每个像素分配类别（天空、道路、行人...）。代表性架构：
- **FCN (Fully Convolutional Network)**：全卷积 + 跳跃连接恢复分辨率
- **U-Net**：编码器-解码器 + 跳跃连接，医学图像金标准

**实例分割**：区分同一类别的不同个体。代表性架构：
- **Mask R-CNN**：Faster R-CNN + 分割分支

### 5.5 生成对抗网络 (GAN)

**核心思想**：两个网络的博弈游戏。

- **生成器 (Generator)** G：从随机噪声生成"假"数据
- **判别器 (Discriminator)** D：判断输入是真实数据还是生成数据

**目标函数**（极小极大博弈）：
```math
min_G max_D 𝔼_{x ∼ p_{data}}[log D(x)] + 𝔼_{z ∼ p_z}[log(1 - D(G(z)))]
```

**训练过程**：
1. 固定 G，训练 D 区分真假（提升判别能力）
2. 固定 D，训练 G 欺骗 D（提升生成质量）
3. 交替进行，直到 D 无法区分 → 生成的数据分布等于真实分布

**为什么 GAN 难以训练？**
- **模式坍塌 (Mode Collapse)**：G 只学会生成少数几种模式，缺乏多样性
- **训练不稳定**：G 和 D 的平衡极难把握
- 解决：WGAN（用 Wasserstein 距离替代 JS 散度）、谱归一化等

## 6. 循环神经网络 (RNN) 与序列模型

> RNN 是处理序列数据的经典架构，其核心思想是**用隐藏状态记录历史信息**。

### 6.1 基本 RNN

#### 6.1.1 原理
```math
h_t = tanh(W_h h_{t-1} + W_x x_t + b)
```
```math
y_t = W_y h_t + b_y
```

**时间展开 (Unrolling)**：RNN 可视为在时间维度上展开的深度网络，每个时间步是一个"层"。

**参数共享**：所有时间步使用相同的权重 W_h, W_x，使得模型可以处理任意长度的序列。

#### 6.1.2 致命缺陷：梯度消失/爆炸

在时间反向传播 (BPTT) 中，梯度需要连乘 W_h：
```math
(∂ 𝓛_T)/(∂ h_t) ∝ ∏_{k=t+1}^T W_h
```

若 W_h 的特征值 < 1 → 梯度指数衰减，长时依赖丢失
若特征值 > 1 → 梯度爆炸

### 6.2 LSTM (长短期记忆网络)

**核心创新**：引入**门控机制**和**记忆细胞**，让网络学会"记住什么"和"忘记什么"。

**三个门**：

1. **遗忘门**：f_t = σ(W_f · [h_{t-1}, x_t] + b_f) — 决定丢弃多少旧信息
2. **输入门**：i_t = σ(W_i · [h_{t-1}, x_t] + b_i) — 决定写入多少新信息
3. **输出门**：o_t = σ(W_o · [h_{t-1}, x_t] + b_o) — 决定输出多少信息

**细胞状态更新**：
```math
c̃_t = tanh(W_c · [h_{t-1}, x_t] + b_c) 候选记忆
```
```math
c_t = f_t ⊙ c_{t-1} + i_t ⊙ c̃_t 状态更新
```
```math
h_t = o_t ⊙ tanh(c_t) 输出
```

**为什么 LSTM 能缓解梯度消失？**

细胞状态 c_t 的梯度包含一条"高速公路"：
```math
(∂ c_t)/(∂ c_{t-1)} = f_t (逐元素乘法)
```

遗忘门 f_t 在 0~1 之间，可以保持接近 1（记住）或接近 0（忘记），使得梯度可以长距离传播。

### 6.3 GRU (门控循环单元)

LSTM 的简化版，将遗忘门和输入门合并为**更新门**：
```math
z_t = σ(W_z · [h_{t-1}, x_t]) 更新门
```
```math
r_t = σ(W_r · [h_{t-1}, x_t]) 重置门
```
```math
h̃_t = tanh(W · [r_t ⊙ h_{t-1}, x_t])
```
```math
h_t = (1 - z_t) ⊙ h_{t-1} + z_t ⊙ h̃_t
```

参数更少，训练更快，效果与 LSTM 相当。

### 6.4 Seq2Seq (序列到序列)

**架构**：编码器-解码器结构。

- **编码器 (Encoder)**：将输入序列 x = (x_1, ..., x_T) 编码为一个固定长度的上下文向量 c
- **解码器 (Decoder)**：从 c 逐步生成输出序列 y = (y_1, ..., y_{T'})

**瓶颈**：所有输入信息必须压缩到单个固定向量 c，长序列效果差 → 注意力机制的动机。

### 6.5 注意力机制 (Attention)

#### 6.5.1 Bahdanau 注意力 (2014)

**核心思想**：解码时不是只看固定的上下文向量，而是**动态地关注编码器中所有位置**，加权求和。
```math
c_t = ∑_{i=1}^T α_{ti} h_i^{enc}
```

注意力权重通过"对齐模型"计算：
```math
α_{ti} = (exp(score(h_t^{dec}, h_i^{enc})))/(∑_{j=1}^T exp(score(h_t^{dec}, h_j^{enc})))
```

这解决了 Seq2Seq 的信息瓶颈问题。而且注意力权重的可视化提供了一定的可解释性。

#### 6.5.2 自注意力 — Transformer 的前奏

自注意力的核心公式（详见第 7 章）：
```math
Attention(Q, K, V) = softmax((QK^T)/(√(d_k)))V
```

## 7. Transformer 与注意力机制

> "Attention Is All You Need" (Vaswani et al., 2017) — 抛弃 RNN 和 CNN，仅依赖注意力机制。这是现代 AI 最重要的架构创新。

### 7.1 Transformer 整体架构
```
 ┌──────────┐
 │ Output │
 └────┬─────┘
 ┌────┴─────┐
 │ Linear │
 │+ Softmax │
 └────┬─────┘
 ┌──────────┴──────────┐
 │ Decoder × N │
 │ ┌──────────────┐ │
 │ │ Masked MHA │ │
 │ │ Cross-Attn │ │
 │ │ FFN │ │
 │ └──────────────┘ │
 └──────────┬──────────┘
 ┌──────────┴──────────┐
 │ Encoder × N │
 │ ┌──────────────┐ │
 │ │ Multi-Head │ │
 │ │ Attention │ │
 │ │ FFN │ │
 │ └──────────────┘ │
 └──────────┬──────────┘
 ┌────┴─────┐
 │ Input │
 │Embedding │
 │+ Pos Enc │
 └──────────┘
```

### 7.2 核心组件详解

#### 7.2.1 自注意力 (Self-Attention)

**原理解析**：

每个 token 的表示通过与序列中所有 token 的交互来更新。

**步骤**：

1. **投影**：对每个 token 的输入表示生成三组向量：
 - Q (Query)："我在找什么？"
 - K (Key)："我有什么信息？"
 - V (Value)："我存储了什么内容？"
```math
Q = XW^Q, K = XW^K, V = XW^V
```

2. **计算注意力分数**：Query 与所有 Key 的点积，衡量"相关性"
```math
S = QK^T
```

3. **缩放**：除以 √(d_k) 防止点积过大导致 Softmax 梯度消失
```math
S_{scaled} = (S)/(√(d_k))
```

4. **Softmax**：将分数转化为概率分布（注意力权重）
```math
A = softmax(S_{scaled})
```

5. **加权求和**：用注意力权重对 Value 加权求和
```math
Attention(Q, K, V) = AV
```

**完整公式**：
```math
Attention(Q, K, V) = softmax((QK^T)/(√(d_k)))V
```

**直观理解**：这是一个**可微的检索-融合操作**。每个 token 发出一系列"查询"，与所有位置的"键"匹配，返回对应"值"的加权和。这模拟了人类阅读时每读一个词都要回顾上下文的行为。

#### 7.2.2 多头注意力 (Multi-Head Attention)

**原理**：单头注意力的表达能力有限。多头注意力让模型同时关注来自不同**表示子空间**的信息。
```math
MultiHead(Q, K, V) = Concat(head_1, ..., head_h)W^O
```
```math
head_i = Attention(QW_i^Q, KW_i^K, VW_i^V)
```

**类比**：一个头可能关注语法结构，另一个关注语义内容，另一个关注指代关系。

#### 7.2.3 位置编码 (Positional Encoding)

**问题**：自注意力是**置换不变的**——交换两个 token 的位置，注意力权重只是交换对应行列，输出也相应交换。模型无法感知 token 的顺序。

**解法**：在输入嵌入中加入位置信息。

**Sinusoidal 位置编码**（原始 Transformer）：
```math
PE(pos, 2i) = sin((pos)/(10000^{2i/d_{model)}})
```
```math
PE(pos, 2i+1) = cos((pos)/(10000^{2i/d_{model)}})
```

**为什么用正弦/余弦？**
- 允许模型轻松学习相对位置关系：PE(pos+k) 可以表示为 PE(pos) 的线性函数
- 能泛化到训练时未见过的序列长度
- 不同维度的波长不同，类似于二进制编码

**可学习位置编码**（BERT/GPT 使用）：更灵活但无法外推到更长的序列。

**RoPE (旋转位置编码)**：目前最流行的方案（Llama, GPT-NeoX 等使用）。通过在 Q、K 上施加旋转变换来编码相对位置，自然地将相对位置信息注入注意力分数。

#### 7.2.4 前馈网络 (FFN)

每个 token 独立地经过相同的两层 MLP：
```math
FFN(x) = GELU(xW_1 + b_1)W_2 + b_2
```

**作用**：
- 为模型提供非线性变换能力（自注意力本质是线性的加权求和）
- 存储知识：FFN 的权重矩阵可以看作键-值记忆（Geva et al., 2021 证明 FFN 第二层相当于在"词汇表"上对第一层激活的"键"进行检索）

#### 7.2.5 残差连接 + Layer Normalization

每个子层（Attention、FFN）都包装在：LayerNorm(x + Sublayer(x)) 中。

- **残差连接**：让梯度直接流通，使深层网络可训练
- **LayerNorm**：稳定训练，加速收敛

**Post-Norm vs Pre-Norm**：
- Post-Norm（原论文）：LN(x + Sublayer(x))，训练不稳定
- Pre-Norm（现代做法）：x + Sublayer(LN(x))，训练更稳定，成为当前标准

### 7.3 复杂度分析

| 操作 | 计算复杂度 | 内存复杂度 |
|------|-----------|-----------|
| Self-Attention | 𝓞(n^2 · d) | 𝓞(n^2) |
| FFN | 𝓞(n · d^2) | 𝓞(n · d) |

n^2 的注意力复杂度是长序列处理的主要瓶颈 → 催生了各种高效注意力变体（FlashAttention, Sparse Attention 等）。

### 7.4 架构变体

**仅编码器 (Encoder-only)** — BERT 系列：
- 双向注意力，适合理解任务（分类、NER、QA）
- 预训练：MLM (Masked Language Modeling)

**仅解码器 (Decoder-only)** — GPT 系列：
- 单向（因果）注意力，适合生成任务
- 预训练：Next Token Prediction
- **这是当前最主流的架构**（GPT-4, Llama, Claude, Gemini 等）

**编码器-解码器** — T5, BART：
- 编码器处理输入，解码器生成输出
- 适合翻译、摘要等 Seq2Seq 任务

## 8. 大语言模型 (LLM)

> LLM 是当前 AI 的核心。理解 LLM 需要理解：预训练 → 微调 → 对齐 → 推理，四个阶段。

### 8.1 LLM 的演进脉络
```
2017: Transformer 论文发表
2018: BERT (双向编码器)、GPT-1 (单向解码器)
2019: GPT-2 (1.5B) — 展示 Zero-shot 能力
2020: GPT-3 (175B) — Few-shot Learning 里程碑
2021: Codex, InstructGPT — 代码生成 + 指令微调
2022: ChatGPT (GPT-3.5) — RLHF 对齐，对话能力惊艳
2023: GPT-4, Llama 2, Claude 2, Mistral — 开源追赶闭源
2024: GPT-4o, Claude 3.5, Gemini, DeepSeek-V2 — 原生多模态
2025: 推理模型 (o1/o3, DeepSeek-R1), 长上下文 (1M+ tokens), Agent 落地
```

### 8.2 预训练 (Pre-training)

#### 8.2.1 训练任务：Next Token Prediction

给定前文 x_{<t} = (x_1, ..., x_{t-1})，预测下一个 token x_t 的概率分布：
```math
P(x_t | x_{<t}) = softmax(W_{vocab} · h_t)
```

**损失函数**（语言建模损失）：
```math
𝓛 = -(1)/(N)∑_{t=1}^N log P(x_t | x_{<t})
```

**为什么这么简单的任务能产生智能？**
- 要准确预测下一个词，模型必须学会：语法、事实知识、推理、常识、甚至是"世界模型"
- 海量数据 + 超大规模模型 → **涌现能力 (Emergent Abilities)**：某些能力在模型规模超过阈值后突然出现

#### 8.2.2 数据与规模

**Scaling Laws** (Kaplan et al., 2020; Chinchilla, 2022)：

最优训练方案是**模型参数和训练 token 数同步增长**。Chinchilla 定律：一个 N 参数的模型最优训练 token 数约为 20N。

也就是说，一个 7B 模型大约需要 140B tokens 的训练数据才能达到最优。

**数据来源**：Common Crawl 网页、书籍、代码 (GitHub)、论文、维基百科等。数据质量比数量更重要。

### 8.3 微调 (Fine-tuning)

#### 8.3.1 全量微调

在预训练模型基础上，用特定任务数据继续训练，更新所有参数。适合有大量标注数据的场景。

#### 8.3.2 指令微调 (Instruction Tuning)

让模型学会遵循指令的关键步骤。收集多样化的 (指令, 回答) 对，用监督学习微调：
```math
𝓛 = -∑_{t} log P(y_t | instruction, y_{<t})
```

**为什么重要？** 预训练模型是"补全引擎"而非"对话助手"。指令微调教会模型将用户意图映射到恰当的回答格式。

#### 8.3.3 参数高效微调 (PEFT)

大模型全量微调成本极高。PEFT 方法只更新极少量参数：

| 方法 | 原理 | 参数量 |
|------|------|--------|
| **LoRA** | 在权重矩阵旁加低秩分解旁路 Δ W = BA | < 1% |
| **Adapter** | 在层间插入小网络 | 2-5% |
| **Prefix Tuning** | 在输入前加可学习的虚拟 token | < 0.1% |
| **QLoRA** | LoRA + 4-bit 量化，GPU 显存需求极小 | < 1% |

**LoRA 原理详解**（最流行的方法）：

冻结预训练权重 W_0 ∈ ℝ^{d × k}，添加低秩分解旁路：
```math
h = W_0 x + Δ W x = W_0 x + BA x
```

其中 B ∈ ℝ^{d × r}, A ∈ ℝ^{r × k}, r ≪ min(d, k)。

r 通常取 8-64，参数增量可忽略不计。低秩假设出奇有效——微调过程中权重变化确实主要集中在低秩子空间。

### 8.4 人类对齐 (Alignment)

#### 8.4.1 RLHF (基于人类反馈的强化学习)

**三阶段**：

1. **SFT (Supervised Fine-Tuning)**：收集人类标注的高质量对话，有监督微调
2. **奖励模型 (Reward Model)**：对同一 prompt 生成多个回答，人类排序；训练模型预测人类偏好分数
3. **PPO 优化**：用强化学习（PPO 算法）最大化奖励模型分数，同时加 KL 惩罚防止偏离太远
```math
𝓛_{PPO} = 𝔼[reward - β · D_{KL}(π_θ ‖ π_{ref})]
```

β 控制对原始策略的偏离程度。

#### 8.4.2 DPO (直接偏好优化)

**动机**：RLHF 需要训练奖励模型 + PPO 优化，流程复杂且不稳定。

**DPO 的创新**：直接在偏好数据上用分类损失优化策略，绕过奖励模型：
```math
𝓛_{DPO} = -𝔼_{(x, y_w, y_l)} [log σ(β log(π_θ(y_w|x))/(π_{ref)(y_w|x)} - β log(π_θ(y_l|x))/(π_{ref)(y_l|x)})]
```

效果相当但更简单稳定，成为当前主流对齐方法之一。

### 8.5 推理优化

#### 8.5.1 解码策略

| 策略 | 原理 | 特点 |
|------|------|------|
| **Greedy** | 每步选概率最高的 token | 确定性，可能陷入重复 |
| **Beam Search** | 维护 K 条候选序列 | 多样性差 |
| **Top-K Sampling** | 从概率最高的 K 个中随机采样 | 增加多样性 |
| **Top-P (Nucleus)** | 从累计概率 ≥ p 的最小 token 集合采样 | 动态调整候选数 |
| **Temperature** | 缩小/放大 logits：p_i ∝ exp(z_i/T) | T < 1 更保守，T > 1 更多样 |

#### 8.5.2 KV Cache

**问题**：每生成一个新 token，都要重新计算所有历史 token 的 Key 和 Value。这导致 O(n^2) 的计算量。

**解决**：缓存已计算的 K、V，每次只对新 token 计算。将推理复杂度从 O(n^2) 降为 O(n)。

#### 8.5.3 FlashAttention

通过分块计算和重新设计内存访问模式，大幅减少 HBM 读写，实现了更快、更省内存的注意力计算。通过 IO 感知的算法设计，使注意力计算的速度接近理论极限。已成标配。

### 8.6 推理模型 (Reasoning Models)

**演进**：o1 (OpenAI, 2024) → o3 → DeepSeek-R1 → Claude 3.5 extended thinking

**核心创新**：让模型在回答前进行**内部思考（链式推理 / Chain-of-Thought）**。

**实现方式**：
- 用强化学习训练模型进行长篇推理
- 推理时分配更多计算资源给"思考"阶段
- 模型学会自我纠错、分解问题、尝试多种解法

**DeepSeek-R1 的关键贡献**：证明了纯 RL（无需 SFT 标注）就能激发出推理行为，且通过蒸馏可以将推理能力迁移到小模型。

## 9. 生成式 AI：图像、视频与多模态

### 9.1 扩散模型 (Diffusion Models)

> 扩散模型是当前图像/视频生成的主流方法，取代了 GAN 的地位。

#### 9.1.1 核心原理

**两个过程**：

1. **前向过程（加噪）**：逐步向真实图像添加高斯噪声，直到变成纯噪声
```math
q(x_t | x_{t-1}) = 𝓝(x_t; √(1-β_t)x_{t-1}, β_tI)
```

其中 β_t 是噪声调度参数，控制每步加噪的量。

2. **反向过程（去噪）**：学习从噪声中逐步恢复图像
```math
p_θ(x_{t-1} | x_t) = 𝓝(x_{t-1}; μ_θ(x_t, t), σ_t^2I)
```

**训练目标**（简化后）：预测添加的噪声
```math
𝓛 = 𝔼_{t, x_0, ε} [‖ε - ε_θ(x_t, t)‖^2]
```

其中 ε_θ 是一个 U-Net（或 DiT）架构的噪声预测网络。

**为什么有效？**
- 将生成任务分解为多个小步骤（通常 50-1000 步），每步都是简单的去噪
- 比 GAN 训练稳定得多，模式覆盖更全面
- 条件控制（文本、图像）只需在噪声预测时注入条件信号

#### 9.1.2 代表性模型

| 模型 | 年份 | 特点 |
|------|------|------|
| **DDPM** | 2020 | 扩散模型的基础框架 |
| **Stable Diffusion** | 2022 | 在潜在空间扩散，大幅降低计算成本 |
| **DALL·E 3** | 2023 | 与 ChatGPT 深度集成，prompt 理解极强 |
| **Midjourney** | 2022- | 美学质量领先 |
| **SDXL / SD3** | 2023-24 | 开源高质量模型，支持多种控制方式 |
| **Flux** | 2024 | 进一步提升了文本跟随和图像质量 |
| **Sora** | 2024 | 视频生成，使用 DiT (Diffusion Transformer) |

#### 9.1.3 Stable Diffusion 关键技术

**潜在扩散 (Latent Diffusion)**：不直接在像素空间操作，而是在 VAE 压缩的低维潜在空间中做扩散，大幅降低计算量。

**条件注入**：通过 Cross-Attention 将文本编码（CLIP Text Encoder）注入去噪过程：
```math
Attention(Q_{latent}, K_{text}, V_{text})
```

**Classifier-Free Guidance (CFG)**：无条件和有条件预测的组合：
```math
ε̂ = ε_θ(x_t, ∅) + w · (ε_θ(x_t, c) - ε_θ(x_t, ∅))
```

w 越大，生成结果越贴合 prompt，但可能牺牲多样性。通常 w=7.5。

**ControlNet**：在 Stable Diffusion 基础上添加可训练的条件控制分支，支持边缘图、深度图、姿态等精确控制。

### 9.2 多模态模型

#### 9.2.1 CLIP — 图文对齐的基石

**原理**：用对比学习将图像和文本映射到同一向量空间。
```math
𝓛 = -(1)/(2N)[∑_i log(exp(sim(I_i, T_i)/τ))/(∑_j exp(sim(I_i, T_j)/τ)) + ∑_i log(exp(sim(T_i, I_i)/τ))/(∑_j exp(sim(T_i, I_j)/τ))]
```

**训练数据**：从互联网收集的 4 亿 (图像, 文本) 对。

**影响**：CLIP 的文本编码器成为 Stable Diffusion 的核心组件，其图像编码器广泛用于各种视觉任务。

#### 9.2.2 多模态大语言模型 (MLLM)

**演进脉络**：
```
Flamingo (2022) → LLaVA (2023) → GPT-4V (2023) → Gemini (2023)
→ GPT-4o / Claude 3.5 (2024) → 原生多模态成为标配
```

**LLaVA 架构**（开源代表）：
```
Image → Vision Encoder (ViT/CLIP) → Projection (MLP) → ┐
 ├→ LLM → Text Output
User Text → Tokenizer → Text Embeddings ────────────────┘
```

视觉编码器提取图像特征，通过一个简单的投影层（通常就是 MLP）映射到 LLM 的输入空间，与文本 token 拼接后一起输入 LLM。

**GPT-4o 的原生多模态**：不再使用独立的编码器-解码器桥接，而是将文本、图像、音频统一处理，实现真正的端到端多模态理解和生成。

## 10. 强化学习

> 强化学习研究智能体如何通过与环境交互来学习最优策略，以最大化累积奖励。

### 10.1 基本框架：马尔可夫决策过程 (MDP)

MDP 由五元组 (𝓢, 𝓐, P, R, γ) 定义：

- 𝓢：状态空间
- 𝓐：动作空间
- P(s'|s, a)：状态转移概率
- R(s, a, s')：奖励函数
- γ ∈ [0, 1]：折扣因子

**交互循环**：
```
State (s_t) → Agent 选择 Action (a_t) → 环境返回 Reward (r_t) + Next State (s_{t+1})
```

**目标**：最大化累积折扣奖励（回报）：
```math
G_t = ∑_{k=0}^{∞} γ^k R_{t+k+1}
```

### 10.2 核心概念

**策略 (Policy)**：π(a|s) 是给定状态 s 下选择动作 a 的概率。

**价值函数 (Value Function)**：从状态 s 出发，按策略 π 行动，能获得的期望回报：
```math
V^π(s) = 𝔼_π[G_t | S_t = s]
```

**Q 函数 (Action-Value Function)**：在状态 s 执行动作 a，之后按策略 π 行动，能获得的期望回报：
```math
Q^π(s, a) = 𝔼_π[G_t | S_t = s, A_t = a]
```

**贝尔曼方程 (Bellman Equation)** — RL 的基石：
```math
V^π(s) = ∑_a π(a|s) ∑_{s'} P(s'|s,a) [R(s,a,s') + γ V^π(s')]
```

直觉：当前状态的价值 = 即时奖励 + 折扣后的下一状态价值。

**最优价值函数**：V^*(s) = max_π V^π(s)。贝尔曼最优方程：
```math
V^*(s) = max_a ∑_{s'} P(s'|s,a) [R(s,a,s') + γ V^*(s')]
```

### 10.3 经典算法

#### 10.3.1 Q-Learning (无模型, Off-Policy)

**核心更新公式**：
```math
Q(s, a) ← Q(s, a) + α [r + γ max_{a'} Q(s', a') - Q(s, a)]
```

- **Off-Policy**：学习最优 Q 值的策略（贪心）与收集数据的策略（ε-greedy）不同
- α 是学习率，TD 误差为 r + γ max_{a'} Q(s', a') - Q(s, a)

**原理**：Q-Learning 利用贝尔曼最优方程，直接从经验中学习最优 Q 函数，无需知道环境模型。

#### 10.3.2 DQN (Deep Q-Network)

将 Q-Learning 与深度学习结合。

**两项关键创新**：

1. **经验回放 (Experience Replay)**：将经验 (s, a, r, s') 存入缓冲区，随机采样训练，打破数据相关性
2. **目标网络 (Target Network)**：使用独立的目标网络 Q̂ 计算 TD 目标，定期同步，稳定训练

**损失函数**：
```math
𝓛 = 𝔼_{(s,a,r,s')} [(r + γ max_{a'} Q̂(s', a'; θ^-) - Q(s, a; θ))^2]
```

**应用**：Atari 游戏达到超越人类水平（直接从像素输入学习）。

#### 10.3.3 策略梯度 (Policy Gradient)

**核心思想**：直接参数化策略 π_θ，通过梯度上升最大化期望回报。

**策略梯度定理**：
```math
∇_θ J(θ) = 𝔼_{π_θ} [∇_θ log π_θ(a|s) · Q^{π_θ}(s, a)]
```

直觉：增加获得高回报动作的概率，降低获得低回报动作的概率。

#### 10.3.4 Actor-Critic

结合策略梯度（Actor）和值函数（Critic）：
- **Actor** (π_θ)：决定怎么做（策略）
- **Critic** (V_φ 或 Q_φ)：评估做得好不好（价值）

**A2C (Advantage Actor-Critic)**：用优势函数 A(s,a) = Q(s,a) - V(s) 替代 Q 值，减小方差。

#### 10.3.5 PPO (Proximal Policy Optimization)

当前最广泛使用的 RL 算法，也是 RLHF 的核心优化器。

**核心思想**：限制策略更新幅度，防止"一步迈太大"导致训练崩溃。

**PPO-Clip 目标函数**：
```math
𝓛^{CLIP}(θ) = 𝔼_t [min(r_t(θ) A_t, clip(r_t(θ), 1-ε, 1+ε) A_t)]
```

其中 r_t(θ) = (π_θ(a_t|s_t))/(π_{θ_{old)}(a_t|s_t)} 是概率比率。

通过裁剪比率来限制更新，PPO 实现了 TRPO 的稳定性和更简单的实现。

### 10.4 RL 在 LLM 中的应用

**RLHF (详见第 8.4 节)**：PPO 优化语言策略，奖励来自人类偏好模型。

**GRPO (DeepSeek-R1 使用)**：Group Relative Policy Optimization。对每个问题采样多个回答，用组内相对质量作为奖励信号，无需训练独立的奖励模型。关键创新是将奖励归一化到组内均值和标准差，使训练更加稳定。

## 11. AI 工程化与 MLOps

### 11.1 训练基础设施

#### 11.1.1 分布式训练

**数据并行 (Data Parallelism)**：每个 GPU 有完整模型副本，处理不同的数据 batch。梯度全局同步（AllReduce）。

**模型并行 (Model Parallelism)**：
- **张量并行**：将一个层的权重矩阵切分到多个 GPU
- **流水线并行**：不同层放到不同 GPU，数据像流水线流过

**ZeRO (DeepSpeed)**：将优化器状态、梯度和参数分布在数据并行组中，而非每个 GPU 存储完整副本。ZeRO-3 分片所有内容，极大地提升了可训练模型规模。

**FSDP (PyTorch)**：PyTorch 官方实现的 ZeRO 类功能。

#### 11.1.2 混合精度训练

**原理**：前向和反向传播使用 FP16/BF16（速度快、省内存），但关键操作（权重更新）使用 FP32（保持精度）。

**损失缩放 (Loss Scaling)**：FP16 的数值范围有限，小梯度可能下溢为 0。将 loss 放大后再反向传播，梯度更新前再缩放回来。

**BF16**：与 FP16 有相同的指数位数但更少的尾数位，数值范围与 FP32 相同，因此不需要损失缩放。逐渐成为主流。

### 11.2 模型压缩与部署

#### 11.2.1 量化 (Quantization)

**原理**：将 32 位浮点参数压缩为更低精度（8-bit、4-bit、甚至 2-bit），大幅减少模型大小和推理延迟。

**GPTQ**：基于近似二阶信息的逐层量化方法。

**AWQ**：观察到只有少量"显著性"权重（~1%）对精度至关重要，对这些权重使用更高精度，大幅提升量化效果。

**GGUF / llama.cpp**：在消费级硬件上运行量化模型的框架，支持 CPU + GPU 混合推理。

#### 11.2.2 知识蒸馏 (Knowledge Distillation)

**原理**：用大模型（Teacher）的输出作为软标签来训练小模型（Student）。
```math
𝓛 = α · 𝓛_{CE}(y, ŷ) + (1-α) · T^2 · D_{KL}(softmax(z_t/T) ‖ softmax(z_s/T))
```

其中 T > 1 是温度参数，用于"软化"概率分布。高温下概率分布更平滑，学生模型可以从中学到类别之间的相似关系——例如猫和狗的相似度远比猫和汽车高。

### 11.3 MLOps 流程
```
数据收集 → 数据验证 → 数据预处理 → 模型训练 → 模型评估
 ↓
 模型注册 → 部署 → 监控 → 反馈
```

### 11.4 模型评估 (Evaluation)

#### 11.4.1 LLM 评测基准

| 基准 | 测试内容 |
|------|----------|
| **MMLU** | 57 个学科的多选题，评估知识广度 |
| **GSM8K** | 小学数学应用题，评估推理能力 |
| **HumanEval** | 代码生成 |
| **MT-Bench** | 多轮对话质量（用 GPT-4 评分） |

| **SWE-bench** | 真实 GitHub issue 的代码修复能力 |

## 12. AI Agent、RAG 与工具使用

### 12.1 RAG (检索增强生成)

#### 12.1.1 核心原理

**问题**：LLM 的知识在训练截止日期后不再更新，且可能产生幻觉（编造事实）。

**解决方案**：在生成回答前，先从外部知识库检索相关信息，将其作为上下文提供给 LLM。
```
用户问题 → 检索 (Retrieve) → 相关文档 → 增强 (Augment) → LLM 生成 → 答案
```

#### 12.1.2 关键组件

**1. 文档处理**：
- 文档切分 (Chunking)：将长文档切分为合适大小的片段（通常 256-1024 tokens）
- 语义切分：基于语义相似度的自适应切分

**2. 向量化 (Embedding)**：
- 使用嵌入模型 (如 text-embedding-3, BGE, Jina) 将文档片段转化为向量

**3. 向量数据库 (Vector DB)**：
- 存储和检索向量：Chroma, Pinecone, Milvus, Weaviate, Qdrant
- 近似最近邻搜索 (ANN)：HNSW, IVF 等算法

**4. 检索策略**：
- 单步检索：一个问题 → 一次检索
- 多步检索：分解问题，迭代检索
- 混合检索：向量检索 + 关键词检索 (BM25)
- 重排序 (Re-ranking)：用更精确（但更慢）的模型对检索结果重新排序

#### 12.1.3 高级 RAG
```
Naive RAG: Query → Retrieve → LLM
Advanced RAG: Query → Rewrite → Retrieve → Rerank → LLM
Agentic RAG: Query → Plan → Multi-step Retrieve → Synthesize → LLM
```

**Self-RAG**：模型在生成过程中自己决定何时检索、检索什么、如何利用检索结果。通过特殊的反思 token 实现。

**Graph RAG**：基于知识图谱而非向量相似度的检索，适合处理实体间复杂关系的查询。

### 12.2 AI Agent

#### 12.2.1 核心概念

**AI Agent** = LLM + 工具 + 规划 + 记忆 + 行动循环
```
┌─────────────────────────────────────┐
│ AI Agent │
│ ┌─────────┐ ┌─────────┐ │
│ │ LLM │ │ Memory │ │
│ │ (Brain) │ │(短期+长期)│ │
│ └────┬────┘ └─────────┘ │
│ │ │
│ ┌────┴────────────────────┐ │
│ │ Planning │ │
│ │ (分解 + 反思) │ │
│ └────────────┬─────────────┘ │
│ │ │
│ ┌────────────┴─────────────┐ │
│ │ Tool Use │ │
│ │ (搜索/代码/API/数据库) │ │
│ └───────────────────────────┘ │
└─────────────────────────────────────┘
```

#### 12.2.2 ReAct 模式

最基本的 Agent 模式：**Re**asoning + **A**cting（推理与行动交替）。
```
Thought: 我需要知道北京今天的天气
Action: search_weather(city="北京")
Observation: 北京今天晴，15-25°C
Thought: 我已经获得了天气信息，可以回答用户了
Answer: 北京今天天气晴朗，温度在 15-25°C 之间
```

#### 12.2.3 规划能力

**任务分解**：将复杂任务分解为子任务。

**Plan-and-Execute**：先制定完整计划，再逐步执行，每步后可反思和调整计划。

**反思 (Reflection)**：Agent 评估自己的输出质量和计划进展，必要时修正。

#### 12.2.4 工具使用 (Function Calling)

**原理**：定义工具的 JSON Schema，LLM 输出结构化的函数调用，系统执行后将结果返回给 LLM。
```
工具定义 → LLM 决定调用哪个工具 + 参数 → 系统执行 → 结果返回 LLM → 生成最终回答
```

**MCP (Model Context Protocol)**：Anthropic 提出的标准化工具接口协议，使得 LLM 可以统一接入各类工具和数据源。

#### 12.2.5 多 Agent 系统

多个专业 Agent 协作，每个负责不同领域：
- 编排者 (Orchestrator) 分配任务
- 专家 Agent 执行各自擅长的子任务
- 通过对话或结构化消息通信

## 13. AI 安全、对齐与伦理

### 13.1 AI 对齐 (Alignment)

**核心问题**：如何确保 AI 系统的目标与人类的价值观和意图一致？

**三大挑战**：
1. **目标错误指定**：奖励函数未能完全捕捉人类的真实偏好
2. **奖励黑客 (Reward Hacking)**：AI 找到捷径最大化奖励但未达成真实目标
3. **分布偏移**：训练和部署环境的变化导致行为偏离预期

### 13.2 安全风险

#### 13.2.1 幻觉 (Hallucination)

LLM 生成看似合理但事实错误的内容。原因：
- 训练数据中的错误信息
- 概率模型的本质（最大化似然 ≠ 事实正确）
- 缺乏可靠的事实校验机制

**缓解方法**：RAG、事实核查、不确定性量化、推理时间搜索。

#### 13.2.2 越狱攻击 (Jailbreaking)

恶意 prompt 设计绕过模型的安全限制。常见类型：
- **角色扮演**："假装你是 DAN (Do Anything Now)..."
- **编码欺骗**：用 Base64 或其他编码隐藏恶意指令
- **多语言攻击**：用小语种绕过安全训练

**防御**：对抗训练、输入/输出过滤、红队测试。

#### 13.2.3 数据隐私

训练数据可能包含个人信息（PII），模型可能在不经意间泄露。解决方案：数据脱敏、差分隐私训练、联邦学习。

### 13.3 伦理与社会影响

- **偏见与歧视**：从有偏见的数据中学到有偏见的模式
- **知识产权**：训练数据版权争议
- **就业冲击**：自动化对劳动力市场的影响
- **信息真实性**：深度伪造 (Deepfake) 和虚假信息传播
- **权力集中**：大模型开发和部署集中在少数科技巨头手中

### 13.4 安全研究前沿

| 方向 | 方法 |
|------|------|
| **机械可解释性** | 理解神经网络的内部工作机制，如特征可视化、电路分析 |
| **红队测试** | 模拟攻击发现漏洞，GPT-4 发布前的系统性红队测试 |
| **Constitutional AI** | 用一系列原则（"宪法"）而非人类反馈来约束模型行为 |
| **Scalable Oversight** | 用 AI 辅助人类监督更强大的 AI |
| **模型评估** | 开发更全面、严格的能力和安全评估基准 |
